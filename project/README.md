# Информационная система строительной компании

Добро пожаловать в проект информационной системы для строительной компании, разработанной по методическим указаниям СКФУ (направление 09.03.04).

## Архитектура

Система реализована по строгой архитектуре **PCMEF** (Presentation → Control → Mediator → Entity → Foundation) с циклическими зависимостями запрещены.

## Технологический стек

- **Java 17**
- **Spring Boot 4.0.6**
- **PostgreSQL**
- **Spring Data JPA**
- **Maven (WAR)**
- **Thymeleaf (SSR)**
- **Spring Security (BCrypt, ROLE_USER/ROLE_ADMIN)**

## Пакетная структура

- `com.skfu.project.control` — слой контроллеров
- `com.skfu.project.mediator` — слой бизнес-логики
- `com.skfu.project.entity` — слой сущностей
- `com.skfu.project.foundation` — слой репозиториев и инфраструктуры
- `com.skfu.project.presentation` — слой Web-контроллеров

## Документация

Вся документация расположена в папке `docs/` и включает:

- **Этап 0: Инициация и бизнес-анализ** — паспорт проекта, диаграммы IDEF0 A-0 и Business Use Cases, глоссарий, SWOT-анализ и матрица стейкхолдеров.
- **Этап 1: Проектирование требований** — функциональные и нефункциональные требования, use case diagrams, прототипы интерфейсов.
- **Этап 2: Архитектурное проектирование** — архитектурные решения, слоистая структура, технологическая архитектура.
- **Этап 3: Проектирование БД** — ER-диаграмма, схема базы данных, SQL-скрипты.
- **Этап 4: Детальное проектирование** — детали реализации компонентов, алгоритмы, интерфейсы.
- **Этап 5: Реализация ядра** — реализация основных модулей, интеграция.
- **Этап 6: Рефакторинг** — улучшение кода, оптимизация.
- **Этап 7: Интерфейс** — разработка UI, интеграция с Thymeleaf.
- **Этап 8: Завершение** — WBS, Гант, COCOMO, ТЗ, Пояснительная записка.
