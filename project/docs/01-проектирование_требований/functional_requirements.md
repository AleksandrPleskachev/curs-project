"# Функциональные требования\n\n## Модуль 1: Управление проектами\n\n### Требование 1.1: Создание проекта\n\n**ID:** FT-PROJ-001\n**Приоритет:** Высокий\n**Стейкхолдер:** Менеджер по работе с клиентами, Руководитель проекта\n\n**Описание:**\nПользователь с ролью ROLE_ADMIN или ROLE_USER может создать новый строительный проект с указанием названия, адреса, бюджета, дат начала и окончания, а также выбрать заказчика из существующих.\n\n**Входные данные:**\n- Название проекта (обязательное, строка до 255 символов)\n- Адрес объекта (обязательное, строка до 500 символов)\n- Бюджет проекта (обязательное, положительное число с двумя десятичными знаками)\n- Дата начала (обязательное, дата в будущем или сегодня)\n- Дата окончания (обязательное, дата после даты начала)\n- Выбор заказчика (обязательное, из списка существующих)\n- Описание (опционально, текстовое поле)\n\n**Действия системы:**\n1. Проверить валидность введенных данных\n2. Проверить наличие прав пользователя (ROLE_ADMIN или ROLE_USER)\n3. Создать запись в таблице `projects`\n4. Назначить текущего пользователя ответственным за проект (если роль USER)\n5. Вернуть сообщение об успешном создании\n\n**Постусловия:**\n- В таблице `projects` появляется новая запись\n- Проект отображается в списке проектов\n\n**Ошибки:**\n- E001: Бюджет должен быть положительным числом\n- E002: Дата окончания должна быть позже даты начала\n- E003: Пользователь не имеет прав на создание проекта\n\n---\n\n### Требование 1.2: Просмотр списка проектов\n\n**ID:** FT-PROJ-002\n**Приоритет:** Высокий\n**Стейкхолдер:** Все пользователи\n\n**Описание:**\nПользователь может просмотреть список всех проектов с возможностью фильтрации по статусу и поиска по названию.\n\n**Входные данные:**\n- Фильтр по статусу (опциональное: Все, В работе, Завершен, Приостановлен)\n- Поиск по названию (опциональное, строка)\n\n**Действия системы:**\n1. Получить список проектов из базы данных\n2. Применить фильтрацию по выбранному статусу\n3. Применить поиск по названию\n4. Отсортировать по дате начала (по убыванию)\n5. Построить пагинацию (10 проектов на страницу)\n\n**Постусловия:**\n- Отображается список проектов на странице\n\n**Ошибки:**\n- E004: Ошибка подключения к базе данных\n\n---\n\n### Требование 1.3: Редактирование проекта\n\n**ID:** FT-PROJ-003\n**Приоритет:** Средний\n**Стейкхолдер:** Руководитель проекта, ROLE_ADMIN\n\n**Описание:**\nПользователь с ролью ROLE_ADMIN или автор проекта может редактировать его параметры.\n\n**Входные данные:**\n- Название проекта\n- Адрес объекта\n- Бюджет проекта\n- Дата начала\n- Дата окончания\n- Статус проекта\n- Описание\n\n**Действия системы:**\n1. Проверить наличие прав пользователя\n2. Валидировать входные данные\n3. Обновить запись в таблице `projects`\n4. Зафиксировать изменения\n\n**Постусловия:**\n- Данные проекта обновлены в базе\n\n**Ошибки:**\n- E005: Проект не найден\n- E006: Пользователь не имеет прав на редактирование\n\n---\n\n## Модуль 2: Управление сметами\n\n### Требование 2.1: Создание сметы\n\n**ID:** FT-EST-001\n**Приоритет:** Высокий\n**Стейкхолдер:** Инженер-сметчик\n\n**Описание:**\nПользователь может создать смету для выбранного проекта, указав позиции с названием, количеством, ценой за единицу и единицей измерения.\n\n**Входные данные:**\n- Выбор проекта (обязательное)\n- Список позиций сметы:\n  - Название позиции (обязательное)\n  - Количество (обязательное, положительное число)\n  - Цена за единицу (обязательное, положительное число)\n  - Единица измерения (обязательное)\n\n**Действия системы:**\n1. Вычислить общую стоимость (сумма всех позиций)\n2. Создать запись в таблице `estimates`\n3. Создать записи в таблице `estimate_items`\n4. Связать смету с проектом\n\n**Постусловия:**\n- В таблице `estimates` появляется новая запись\n- В таблице `estimate_items` создаются позиции\n\n**Ошибки:**\n- E007: Проект не найден\n- E008: Количество должно быть положительным числом\n\n---\n\n## Модуль 3: Управление задачами\n\n### Требование 3.1: Создание задачи\n\n**ID:** FT-TASK-001\n**Приоритет:** Высокий\n**Стейкхолдер:** Руководитель проекта, ROLE_ADMIN\n\n**Описание:**\nПользователь может создать задачу для проекта, назначить исполнителя и установить сроки.\n\n**Входные данные:**\n- Выбор проекта (обязательное)\n- Название задачи (обязательное)\n- Описание (опциональное)\n- Приоритет (обязательное: Низкий, Средний, Высокий, Критический)\n- Исполнитель (обязательное, выбор из сотрудников проекта)\n- Плановая дата начала (обязательное)\n- Плановая дата окончания (обязательное, после даты начала)\n\n**Действия системы:**\n1. Проверить права пользователя\n2. Валидировать даты\n3. Создать запись в таблице `tasks`\n4. Связать с проектом и исполнителем\n\n**Постусловия:**\n- В таблице `tasks` появляется новая запись\n\n**Ошибки:**\n- E009: Исполнитель не найден в сотрудниках проекта\n- E010: Дата окончания должна быть позже даты начала\n\n---\n\n## Модуль 4: Управление сотрудниками\n\n### Требование 4.1: Добавление сотрудника к проекту\n\n**ID:** FT-EMP-001\n**Приоритет:** Средний\n**Стейкхолдер:** Руководитель проекта, ROLE_ADMIN\n\n**Описание:**\nПользователь может добавить сотрудника к проекту, назначив ему роль и набор задач.\n\n**Входные данные:**\n- Выбор сотрудника (из списка всех сотрудников компании)\n- Роль в проекте (обязательное: Руководитель, Инженер, Сметчик, Другое)\n\n**Действия системы:**\n1. Проверить права пользователя\n2. Проверить, не добавлен ли сотрудник уже к проекту\n3. Создать связь в промежуточной таблице `project_employees`\n4. Назначить задачи (опционально)\n\n**Постусловия:**\n- Сотрудник добавлен к проекту\n- В таблице `project_employees` появляется новая запись\n\n**Ошибки:**\n- E011: Сотрудник уже добавлен к проекту\n- E012: Сотрудник не найден\n\n---\n\n## Модуль 5: Управление пользователями и ролями\n\n### Требование 5.1: Аутентификация\n\n**ID:** FT-USR-001\n**Приоритет:** Высокий\n**Стейкхолдер:** Все пользователи\n\n**Описание:**\nПользователь может войти в систему с помощью email и пароля.\n\n**Входные данные:**\n- Email (обязательное)\n- Пароль (обязательное)\n\n**Действия системы:**\n1. Найти пользователя по email\n2. Проверить пароль с помощью BCrypt\n3. Создать сессию\n4. Перенаправить на страницу профиля или dashboard\n\n**Постусловия:**\n- Пользователь аутентифицирован\n- В сессии сохранены данные пользователя\n\n**Ошибки:**\n- E013: Пользователь не найден\n- E014: Неверный пароль\n\n---\n\n### Требование 5.2: Ролевая модель\n\n**ID:** FT-USR-002\n**Приоритет:** Высокий\n**Стейкхолдер:** ROLE_ADMIN\n\n**Описание:**\nСистема поддерживает две роли: ROLE_USER и ROLE_ADMIN. ROLE_ADMIN имеет доступ ко всем функциям, ROLE_USER — только к личным задачам и своим проектам.\n\n**Роли:**\n- **ROLE_ADMIN:** Полный доступ ко всем модулям, управление пользователями\n- **ROLE_USER:** Просмотр своих задач, проектов, где он участвует, создание задач для себя\n\n**Ограничения:**\n- ROLE_USER не может создавать/редактировать/удалять пользователей\n- ROLE_USER видит только свои задачи\n\n---\n\n## Матрица приоритетов\n\n| ID | Название | Приоритет | Комментарий |\n|----|----------|-----------|-------------|\n| FT-PROJ-001 | Создание проекта | Высокий | Базовый функционал |\n| FT-PROJ-002 | Список проектов | Высокий | Базовый функционал |\n| FT-EST-001 | Создание сметы | Высокий | Критичный для бизнеса |\n| FT-TASK-001 | Создание задачи | Высокий | Основная функция |\n| FT-USR-001 | Аутентификация | Высокий | Обязательный модуль |\n| FT-USR-002 | Ролевая модель | Высокий | Безопасность |\n| FT-PROJ-003 | Редактирование проекта | Средний | Удобство |\n| FT-EMP-001 | Добавление сотрудника | Средний | Удобство |\n"
