"# Нефункциональные требования\n\n## Производительность\n\n### NFR-PERF-001: Ответ времени\n\n**Требование:**\nВремя отклика системы на действия пользователя не должно превышать:\n- 2 секунды для загрузки страницы списка (до 100 записей)\n- 3 секунды для загрузки детальной страницы (с JOIN FETCH)\n- 5 секунд для формирования отчета (до 1000 записей)\n\n**Методы измерения:**\n- Замер времени загрузки страницы через браузер\n- Мониторинг времени выполнения SQL-запросов\n\n**Методы достижения:**\n- Индексация полей в базе данных\n- Использование JOIN FETCH для избежания N+1\n- Кэширование часто запрашиваемых данных\n\n---\n\n### NFR-PERF-002: Параллелизм\n\n**Требование:**\nСистема должна поддерживать одновременную работу не менее 50 пользователей без деградации производительности.\n\n**Методы достижения:**\n- Использование пулла соединений (HikariCP)\n- Оптимизация SQL-запросов\n- Горизонтальное масштабирование при необходимости\n\n---\n\n## Безопасность\n\n### NFR-SEC-001: Аутентификация и авторизация\n\n**Требование:**\nВсе пользователи должны проходить аутентификацию. Доступ к функциям регулируется через ролевую модель (ROLE_USER, ROLE_ADMIN).\n\n**Методы достижения:**\n- Spring Security с BCrypt для хранения паролей\n- Аннотации `@PreAuthorize` для ограничения доступа к методам\n- Проверка прав в контроллерах и шаблонах\n\n**Матрица ролей:**\n- ROLE_USER: Доступ к своим задачам, проектам\n- ROLE_ADMIN: Полный доступ ко всем функциям\n\n---\n\n### NFR-SEC-002: Защита от SQL-инъекций\n\n**Требование:**\nСистема должна быть защищена от SQL-инъекций.\n\n**Методы достижения:**\n- Использование JPA/Hibernate (PreparedStatement)\n- Валидация входных данных\n- Использование nativeQuery с осторожностью\n\n---\n\n### NFR-SEC-003: Безопасность сессий\n\n**Требование:**\nСессии пользователей должны быть защищены от перехвата.\n\n**Методы достижения:**\n- Использование HTTPS\n- Установка флага `secure` для cookies\n- Таймаут сессии (30 минут неактивности)\n\n---\n\n## Надежность\n\n### NFR-REL-001: Непрерывная работа\n\n**Требование:**\nСистема должна быть доступна 99% времени.\n\n**Методы достижения:**\n- Регулярное резервное копирование базы данных\n- Мониторинг состояния сервера\n- Логирование ошибок\n\n---\n\n### NFR-REL-002: Обработка ошибок\n\n**Требование:**\nВсе ошибки должны логироваться, пользователь должен получать понятное сообщение.\n\n**Методы достижения:**\n- GlobalExceptionHandler с `@ControllerAdvice`\n- Логирование через SLF4J\n- Отображение сообщений об ошибках в шаблонах\n\n---\n\n## Удобство использования\n\n### NFR-UsE-001: Интуитивность\n\n**Требование:**\nИнтерфейс должен быть понятен пользователю без инструкции.\n\n**Методы достижения:**\n- Использование стандартных UI-элементов\n- Подсказки (tooltips) для неочевидных элементов\n- Консистентность названий кнопок и действий\n\n---\n\n### NFR-UsE-002: Адаптивность\n\n**Требование:**\nИнтерфейс должен корректно отображаться на экранах с разрешением от 1024x768.\n\n**Методы достижения:**\n- Использование Bootstrap CSS\n- Проверка на различных устройствах\n\n---\n\n## Масштабируемость\n\n### NFR-SCA-001: Расширяемость\n\n**Требование:**\nСистема должна поддерживать добавление новых модулей без переписывания существующего кода.\n\n**Методы достижения:**\n- Слабая связанность модулей\n- Использование интерфейсов\n- Принцип единственной ответственности\n\n---\n\n### NFR-SCA-002: Горизонтальное масштабирование\n\n**Требование:**\nСистема должна поддерживать горизонтальное масштабирование.\n\n**Методы достижения:**\n- Использование stateless архитектуры\n- Централизованное хранилище сессий (Redis)\n- Балансировка нагрузки\n\n---\n\n## Поддерживаемость\n\n### NFR-SUP-001: Тестируемость\n\n**Требование:**\nКод должен быть тестируемым.\n\n**Методы достижения:**\n- Использование интерфейсов для сервисов\n- DI (Dependency Injection)\n- Модульные тесты (JUnit + Mockito)\n\n---\n\n### NFR-SUP-002: Документированность\n\n**Требование:**\nКод должен быть документирован.\n\n**Методы достижения:**\n- JavaDoc для публичных методов\n- Комментарии для сложной логики\n- Документация в `docs/`\n\n---\n\n## Матрица приоритетов\n\n| ID | Название | Приоритет | Комментарий |\n|----|----------|-----------|-------------|\n| NFR-PERF-001 | Время отклика | Высокий | Критично для UX |\n|
