# Описание Use Case диаграммы

**Акторы:**
*   **Менеджер:** Управляет заказчиками, проектами, назначает задачи.
*   **Сметчик:** Создает и редактирует сметы.
*   **Руководитель проекта:** Контролирует прогресс проекта, утверждает сметы и задачи.
*   **Заказчик:** Просматривает статус своего проекта.
*   **Система:** Внутренний актор, отвечающий за уведомления и фоновые задачи.

**Прецеденты (Use Cases):**

*   **Управление заказчиками (Менеджер)**
    *   Добавить нового заказчика
    *   Просмотреть список заказчиков
    *   Редактировать данные заказчика
    *   Удалить заказчика

*   **Управление проектами (Менеджер, Руководитель проекта)**
    *   Создать новый проект
    *   Просмотреть список проектов
    *   Просмотреть детали проекта
    *   Обновить статус проекта
    *   Закрыть проект

*   **Управление сметами (Сметчик, Руководитель проекта)**
    *   Создать смету для проекта
    *   Рассчитать стоимость сметы
    *   Отправить смету на утверждение
    *   Утвердить смету
    *   Просмотреть историю смет

*   **Управление задачами (Менеджер, Руководитель проекта)**
    *   Создать задачу для проекта
    *   Назначить задачу сотруднику
    *   Обновить статус задачи
    *   Просмотреть прогресс выполнения задач

*   **Управление договорами (Менеджер)**
    *   Создать договор
    *   Подписать договор
    *   Зафиксировать оплату по договору

*   **Формирование отчетов (Руководитель проекта, Система)**
    *   Сформировать отчет о ходе строительства
    *   Сформировать финансовый отчет
    *   Сформировать отчет по использованию оборудования

*   **Просмотр статуса проекта (Заказчик)**
    *   Авторизоваться в системе
    *   Просмотреть прогресс своего проекта
    *   Просмотреть назначенные задачи

*   **Уведомления (Система)**
    *   Отправить уведомление о приближающемся сроке задачи
    *   Отправить уведомление об утверждении сметы

Для построения диаграммы используйте инструмент draw.io или PlantUML, разместив акторов и связав их с прецедентами. Группируйте прецеденты по акторам.
