# COCOMO - Оценка трудоемкости разработки

## Введение

Методика COCOMO (Constructive Cost Model) - это эмпирическая модель оценки трудоемкости разработки программного обеспечения. В данном расчете используется базовая модель COCOMO с учетом характеристик проекта.

---

## Исходные данные

| Параметр | Значение | Комментарий |
|----------|----------|-------------|
| KLOC (Количество тысяч строк кода) | 12.5 | Оценка по итогам реализации |
| Модель проекта | Органический | Небольшая команда, опытная |
| Параметр a | 3.2 | Для органических проектов |
| Параметр b | 1.05 | Для органических проектов |
| EAF (Фактор оценки effort) | 1.0 | Упрощенная модель |

---

## Расчет по базовой модели COCOMO

### Формула

```
Effort = a × (KLOC)^b
```

### Подстановка значений

```
Effort = 3.2 × (12.5)^1.05
Effort = 3.2 × 14.62
Effort = 46.79 человеко-месяцев
```

## Факторы, повлиявшие на трудоемкость

### Положительные факторы (снижение трудозатрат)
- ✅ Наличие опытного руководства
- ✅ Использование Spring Boot (быстрая разработка)
- ✅ Наличие аналогов для изучения
- ✅ Четкое понимание требований
- ✅ Эффективная организация работы

### Отрицательные факторы (повышение трудозатрат)
- ⚠️ Недостаток опыта в Spring Security
- ⚠️ Проблемы с кодировкой при работе с JaCoCo
- ⚠️ Необходимость изучения Docker
