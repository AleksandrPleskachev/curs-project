# ADR: Выбор архитектуры PCMEF

**Дата:** 01.06.2026

**Статус:** Принято

## Контекст
Для обеспечения масштабируемости, тестируемости и поддерживаемости приложения необходимо выбрать подходящую архитектуру. Проект требует строгого разделения ответственностей и предотвращения циклических зависимостей между компонентами.

## Решение
Было принято решение использовать архитектурный шаблон **PCMEF** (Presentation → Control → Mediator → Entity → Foundation). Этот шаблон является вариацией слоистой архитектуры, адаптированной под требования проекта.

## Альтернативы
1.  **Традиционная слоистая архитектура (Presentation, Business, Data):** Более простая, но может привести к "толстым" сервисным слоям и смешению ответственностей.
2.  **Чистая архитектура (Clean Architecture):** Очень гибкая, но может быть избыточной для данного проекта и требует больше усилий на настройку.
3.  **Микросервисы:** Позволяет независимое развертывание, но значительно усложняет инфраструктуру и управление, что не оправдано для монолитного приложения.

## Последствия
**Положительные:**
*   Четкое разделение ответственностей между слоями.
*   Упрощение тестирования (возможность мокирования на уровне Mediator и Foundation).
*   Предотвращение циклических зависимостей благодаря строгой направленности.
*   Легкость внесения изменений в отдельные слои.

**Отрицательные:**
*   Увеличение количества классов и пакетов, что может усложнить навигацию в начале.
*   Необходимость строгого соблюдения правил зависимостей разработчиками.
*   Небольшая накладная стоимость на вызовы между слоями.
